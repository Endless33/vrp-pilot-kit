# Pilot Requirements

## Purpose

This document defines the minimum engineering requirements for participation in the VRP Pilot Program.

The objective is to ensure that Pilot environments are suitable for meaningful technical evaluation.

---

## Supported Environments

Typical Pilot environments may include:

- Physical servers
- Virtual machines
- Cloud infrastructure
- Laboratory environments
- Development environments
- Staging environments

Production deployment is evaluated separately.

---

## Operating Systems

Typical supported platforms include:

- Linux
- Oracle Linux
- Ubuntu Server
- Debian
- Rocky Linux
- Windows Server

Additional platforms may be evaluated individually.

---

## Network Access

Pilot environments should provide:

- stable Internet connectivity
- administrative access
- ability to capture logs
- firewall configuration when required
- outbound connectivity for approved services

---

## Engineering Contact

Each Pilot should designate at least one engineering contact responsible for:

- deployment
- troubleshooting
- validation
- communication

---

## Security

Pilot participants should:

- protect deployment credentials
- restrict unnecessary access
- follow internal security policies
- avoid exposing Pilot environments publicly unless agreed

---

## Validation

Pilot participants should be prepared to perform:

- installation verification
- functional validation
- integration testing
- operational observation
- engineering reporting

---

## Reporting

Participants should record:

- deployment observations
- unexpected behavior
- compatibility issues
- validation results
- engineering feedback

---

## Commercial Runtime

Protected Runtime components are provided only as part of an approved Pilot engagement.

They are not distributed publicly.

---

## Engineering Principle

Pilot readiness is measured by engineering preparedness rather than infrastructure size.

Engineering continues.
