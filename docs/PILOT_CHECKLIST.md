# Pilot Checklist

## Purpose

This checklist is intended to help Pilot participants prepare a structured engineering evaluation before deployment.

Completion of this checklist improves reproducibility and reduces deployment delays.

---

## Organization

- [ ] Pilot application submitted
- [ ] Technical contact assigned
- [ ] Evaluation objectives defined
- [ ] Internal approval completed

---

## Infrastructure

- [ ] Target environment prepared
- [ ] Required operating system installed
- [ ] Administrative access confirmed
- [ ] Internet connectivity verified
- [ ] Firewall rules reviewed
- [ ] DNS configuration verified

---

## Security

- [ ] Access credentials secured
- [ ] Internal security review completed
- [ ] Backup procedure available
- [ ] Logging enabled
- [ ] Monitoring available

---

## Engineering

- [ ] Validation scope agreed
- [ ] Success criteria documented
- [ ] Test scenarios prepared
- [ ] Evidence collection planned
- [ ] Engineering contacts available

---

## Deployment

- [ ] Runtime package received
- [ ] Deployment instructions reviewed
- [ ] Configuration verified
- [ ] Initial startup completed
- [ ] Health verification completed

---

## Validation

- [ ] Functional validation completed
- [ ] Compatibility verified
- [ ] Expected behavior confirmed
- [ ] Engineering observations recorded
- [ ] Evidence collected

---

## Final Review

- [ ] Engineering report completed
- [ ] Questions documented
- [ ] Improvement suggestions prepared
- [ ] Final evaluation meeting scheduled
