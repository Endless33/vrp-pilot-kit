# Security Policy

## Purpose

This document describes the public security policy for the VRP pilot repository.

It explains how publicly observable security issues should be reported and how security-related information is handled within this repository.

This repository contains public documentation and integration materials only.

It does not contain the protected VRP runtime implementation.

---

## Scope

This policy applies to:

- public documentation;
- deployment guidance;
- integration contracts;
- validation materials;
- example scripts;
- report templates;
- public interfaces.

It does not apply to:

- protected runtime implementation;
- confidential engineering processes;
- private pilot environments;
- proprietary runtime algorithms.

---

## Reporting Security Issues

If you discover a security-related issue affecting the public pilot materials, please provide:

- a clear description of the issue;
- reproduction steps where applicable;
- affected documentation or public component;
- supporting evidence.

Reports should focus on observable behavior.

Please do not include confidential information.

---

## Responsible Disclosure

Security observations should be reported responsibly.

Public disclosure before the issue has been reviewed is discouraged when it could unnecessarily expose participants or create confusion.

---

## Out of Scope

The following are outside the scope of this repository:

- requests for protected runtime source code;
- proprietary implementation details;
- confidential operational procedures;
- speculative analysis of undisclosed implementation.

---

## Repository Security Principles

This repository is maintained according to the following principles.

- Public documentation only.
- No protected implementation.
- Observable behavior over implementation details.
- Evidence-based technical discussion.
- Responsible disclosure.

---

## Supported Materials

Security updates may be published for:

- documentation;
- deployment guidance;
- validation procedures;
- example artifacts;
- public interfaces.

Older documentation may remain available for historical reference.

---

## Confidentiality

Do not publish:

- credentials;
- private infrastructure information;
- protected runtime implementation;
- proprietary engineering documentation.

---

## Summary

The VRP public repository supports responsible reporting of security issues affecting publicly available materials while preserving the confidentiality of the protected VRP runtime implementation.
