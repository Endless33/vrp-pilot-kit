# Code of Conduct

## Purpose

This repository exists to support technical discussion, public pilot documentation, and independent engineering evaluation of the VRP pilot.

All participants are expected to contribute in a professional, respectful, and evidence-based manner.

---

## Our Principles

We encourage contributions that are:

- technically accurate;
- constructive;
- respectful;
- reproducible;
- supported by observable evidence.

Discussion should focus on engineering rather than individuals.

---

## Expected Behavior

Contributors are encouraged to:

- communicate respectfully;
- provide constructive technical feedback;
- report reproducible issues;
- support engineering discussions with evidence;
- improve documentation through well-reasoned contributions.

---

## Unacceptable Behavior

The following behavior is not appropriate within this repository:

- personal attacks;
- harassment;
- discrimination;
- abusive language;
- intentionally misleading technical information;
- spam or promotional content unrelated to the repository;
- disclosure of confidential or protected implementation details.

---

## Technical Discussions

Engineering disagreements are expected.

When disagreements occur, contributors should:

- explain their reasoning;
- provide supporting evidence where possible;
- reference documented behavior;
- remain respectful throughout the discussion.

Technical merit should take precedence over opinion.

---

## Confidentiality

This repository documents only the public VRP pilot boundary.

Contributors should not submit:

- protected runtime source code;
- proprietary algorithms;
- confidential engineering documentation;
- private operational procedures;
- internal implementation details.

---

## Enforcement

Repository maintainers may:

- request clarification;
- request revisions;
- close discussions;
- remove inappropriate content;
- restrict participation when necessary to preserve constructive collaboration.

---

## Scope

This Code of Conduct applies to:

- issues;
- pull requests;
- discussions;
- documentation contributions;
- all public interactions associated with this repository.

---

## Summary

The goal of this Code of Conduct is to maintain a professional environment where engineering discussions remain respectful, technically grounded, and focused on improving the public VRP pilot documentation.
