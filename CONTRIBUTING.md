# Contributing

## Purpose

Thank you for your interest in contributing to the VRP public pilot repository.

This document explains how contributions to the public documentation, deployment guidance, validation materials, and supporting examples should be prepared and submitted.

The repository documents only the public pilot boundary.

Protected runtime implementation is intentionally excluded.

---

## Scope

Contributions are welcome for:

- documentation;
- deployment guidance;
- validation documentation;
- report templates;
- example artifacts;
- example scripts;
- public integration materials.

Contributions should not include:

- protected runtime source code;
- proprietary algorithms;
- confidential engineering documentation;
- internal implementation details.

---

## Before Contributing

Please review:

- README.md
- Architecture Overview
- Integration Boundary
- Validation Model
- Evidence Model
- Public Versioning Model

Understanding the existing documentation helps maintain consistency.

---

## Types of Contributions

Examples include:

- documentation improvements;
- clarification of existing content;
- correction of technical errors;
- improved examples;
- formatting improvements;
- reproducibility improvements.

---

## Contribution Principles

Every contribution should strive to be:

- technically accurate;
- reproducible;
- well documented;
- evidence-based;
- consistent with existing terminology.

---

## Pull Requests

Before opening a pull request:

- review the existing documentation;
- verify technical accuracy;
- ensure formatting is consistent;
- avoid unrelated changes;
- complete the pull request template.

---

## Documentation Style

Documentation should:

- use clear engineering language;
- avoid unnecessary ambiguity;
- distinguish observable behavior from implementation;
- preserve the public security boundary.

---

## Security

Do not contribute:

- protected runtime implementation;
- confidential operational information;
- proprietary engineering methods;
- private runtime state.

If uncertain, omit the information and request clarification.

---

## Review Process

Public contributions may be reviewed for:

- technical correctness;
- documentation quality;
- consistency;
- compatibility;
- reproducibility.

Additional revisions may be requested before acceptance.

---

## Licensing

By submitting a contribution, you confirm that you have the right to contribute the submitted material under the repository license.

---

## Summary

Contributions should strengthen the quality, clarity, and reproducibility of the public VRP pilot documentation while preserving the separation between public engineering materials and the protected runtime implementation.
